# 个人工作台（Personal Workbench）

一个**纯前端、单文件**的个人效率工作台，覆盖 **项目管理 / 科研 / 生活 / 运动 / 金融** 五大板块。打开即用，**无需后端、无需数据库**。

> 定位演进：**v2 是「记录器」，v3 开始成为「教练」**——核心变化不在"能不能记"，而在"记完之后有没有人帮你分析、提醒、复盘、养成习惯"。
> 北极星指标建议：`周活跃复盘率`（每周至少打开一次复盘并标记 ≥1 项完成）。

## ✨ v3 相对 v2 的改进

### 一、通用增强

| 模块 | v2 | v3 | 价值 |
|------|----|----|------|
| 复盘 | 无（概览只是当下快照） | **新增「📈 复盘」页**：自动聚合本周完成/逾期/停滞/下周待办/运动达标/收支结余/习惯打卡 | 补齐"周日 10 分钟复盘"仪式感 |
| 习惯 | 仅"运动周计划"（弱代理） | **新增「🔥 习惯」一等模块**：每日/每周打卡、连续天数、近 7 天格子、CRUD | 读书/记账/冥想等习惯有了归属 |
| 财务 | 有分类字段 + 列表式分类汇总 | **预置分类 + SVG 环形图占比 + 储蓄率卡 + 月度预算卡** | 从"知道花哪了"升级到"知道健不健康" |
| 标签 | 无（只有 cat + 项目维度） | **事项支持 tags**，卡片显示 `#标签`，搜索支持 `tag:` 前缀 | 灵活分组（重点/等回复）、跨项目检索 |
| 项目健康度 | 仅里程碑/任务进度条 | **健康灯（绿/黄/红）+ 逾期任务数 + 风险里程碑 + 工时估算准确率** | 一眼看出哪个项目在着火；用 est vs act 反哺排期 |
| 年度热力图 | 无 | **概览新增 GitHub 式半年度活跃热力图** | 用"连续天数"制造正反馈，强化坚持感 |
| 完成时间 | 仅状态标记 | 事项 `done` 时写入 `completedAt` | 复盘/热力图才有真实时间锚点 |

### 二、🔬 科研专项增强（A1 + A2）

#### A1 · 审稿回复截止日 + 轮次（直击在审痛点）
- 论文表单新增 **「审稿回复截止日」**（日期）+ **「审稿轮次」**（R1–R4）。
- 论文卡片显示 `#R2` 紫色标签与 `⏰ 回复还剩 X 天` 徽章（逾期红 / ≤30 天橙 / 更远蓝）。
- **科研 tab 顶部新增「⏰ 审稿回复倒计时」横幅**：自动列出所有有回复截止、且未录用/未拒稿的论文，按截止日排序——打开科研页第一眼就能看到还差几天。

#### A2 · Obsidian 一键打开（含精确配置）
- 论文 / 专利 / 项目卡片各加 **📓 按钮**，唤起本机 Obsidian 并定位对应笔记（本地协议 `obsidian://`，不上传数据）。
- 点 📓 判定优先级：
  1. 填了完整 `obsidian://` 链接 → **精确打开**该笔记（最快最准）；
  2. 只填了笔记路径（如 `科研/CVPR2026_response`）→ 自动拼 `obsidian://open?file=...` 用默认 vault 打开；
  3. 留空 → **降级为按标题模糊搜索**（兼容旧行为）。
- 配置为**可选**：想精确就粘链接，不想管就留空继续模糊搜。
- 详细接入与排查见 [`个人工作台v3_Obsidian接入说明.md`](个人工作台v3_Obsidian接入说明.md)。

### 三、数据模型变更（完全兼容 v2）
| 字段 | 说明 | 兼容处理 |
|------|------|----------|
| `data.habits` | 新数组 | `load()` / `restoreBak()` / 同步 / storage 事件四处均加 `if(!data.habits)data.habits=[]`，导入旧 v2 备份不丢数据 |
| `item.completedAt` | 完成时间戳 | `toggle()` 与 `submitForm()` 写入；旧数据无此字段时回退到 `due` |
| `item.tags` | 标签数组 | 旧数据无 tags，`(i.tags||[])` 全部兜底为空 |

导入/导出 JSON、本地存储、多标签同步、主题、备份还原等 v2 能力**全部保留**，v3 为纯增量改造。

## 💾 数据存储与同步

- 默认数据保存在本机浏览器的 `localStorage`。
- 支持 **GitHub Gist 同步**（在「☁ 同步设置」中配置），方便多设备备份。
- 支持 **本地备份与回滚**，防止数据丢失。

> 提示：清除浏览器数据会清空本地记录，建议定期通过 Gist 或本地备份导出。

## 🌐 部署为在线网页（可选）

如需把它当成可访问的在线页面，可在仓库 **Settings → Pages** 中启用 GitHub Pages（源分支 `main`，根目录 `/`），即可获得类似 `https://<用户名>.github.io/personal-workbench/` 的访问地址。
