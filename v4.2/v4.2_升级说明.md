# 个人工作台 · v4.1 → v4.2 升级说明

> v4.2 基于 v4.1 建立，本阶段重点把**生活管理模块**从「记录」升级为「行动化」。默认导航不变，仍只保留 今天 / 工作 / 日历 / 回顾 / 更多 五个核心 Tab。

## 一、生活模块：从「记录」升级为「行动化」

### 1.1 新增「生活首页」

v4.1 的生活是事项 + 书籍 + 旅行 + 纪念日的列表式收纳。v4.2 新增「生活首页」，优先回答四个问题：

| 区块 | 优先展示 |
|---|---|
| 今天事项 | 今天和已逾期的生活事项 |
| 下一段出行 | 最近的出行安排 |
| 近期重要日子 | 接下来的纪念日/重要日子 |
| 当前阅读 | 正在阅读的书 |

### 1.2 事项分组（移除模块内重复日历）

v4.1 的生活事项是平铺列表，并且模块内还有重复日历。v4.2 改为按时间分组：

- 今天与已到期
- 未来 7 天
- 暂无日期
- 更晚
- 已完成

模块内重复日历移除（日历的功能统一收回到「日历」核心 Tab）。

### 1.3 出行清单升级

- **逐项勾选**：清单支持逐项打勾，并显示准备进度（如「3/5 已完成」）
- **下一步行动**：清单页显示当前要做的下一步
- **财务关联**：财务记录可关联出行，出行页自动汇总已发生的关联支出
- **默认收起**：已结束的出行默认收起，减少长期使用后的页面膨胀

### 1.4 阅读升级

| 改动 | 说明 |
|---|---|
| 下一步行动 | 阅读新增「下一步行动」字段，不再只是状态标记 |
| 计划日期 | 可设置计划开始/读完日期 |
| 快捷动作 | 支持「开始阅读」「进度 +10%」「标记读完」 |

### 1.5 纪念日 → 「重要日子」

- 纪念日扩展为更通用的「重要日子」
- 支持**提前 7 / 14 / 30 天**准备
- 一键生成对应的生活事项（如「准备生日礼物」）

## 二、页面膨胀治理

- 已结束出行默认收起
- 已完成生活事项默认收起
- 减少长期使用后的视觉负担

## 三、数据兼容

- 完全兼容 v4.1 的生活事项、书籍、旅行清单、纪念日
- 完全兼容 v4.1 的手填出行花费
- 老数据无需迁移，直接打开 v4.2 即可使用

## 四、测试覆盖

新增 `tests/life-summary.test.js`，覆盖：

- 事项分组（5 类分组逻辑）
- 旧旅行清单迁移（向后兼容）
- 财务关联（出行自动汇总关联支出）
- 重要日子（提前准备天数 + 一键生成事项）
- 生活首页渲染

至此 v4.2 共有 5 个测试套件：

| 测试文件 | 覆盖点 |
|---|---|
| `tests/module-preferences.test.js` | 模块启用、导航拦截、置顶上限、数据保留 |
| `tests/research-summary.test.js` | 论文阶段统计、等待/跟进优先级、跨类型截止日汇总 |
| `tests/health-metrics.test.js` | 个性化目标、周进度、跳过计划、身体趋势计算 |
| `tests/finance-metrics.test.js` | 实际与计划分离、预算、分类及趋势统计 |
| `tests/life-summary.test.js` | 事项分组、旧旅行清单迁移、财务关联、重要日子与首页渲染 |

## 五、迁移建议

1. **数据兼容**：v4.2 直接读取 v4.1 的 localStorage 数据，无需迁移
2. **首次打开**：进入「生活」模块后默认展示「生活首页」
3. **首要体验**：去「生活首页」看看今天事项、下一段出行和当前阅读——这是 v4.1 没有的视角
4. **构建自定义便携版**：修改 `source/src/` 后运行 `python scripts/build_portable.py` 重新生成
