# 个人工作台 · v3 → v4 升级说明

> v4 从「多功能管理后台」收敛为「简洁、本地优先的个人助手」：默认导航只保留 **今天 / 工作 / 日历 / 回顾 / 更多**，科研、生活、健康、习惯、财务和热榜改为按需启用。

## 一、整体定位的转变

| 维度 | v3 | v4 |
|---|---|---|
| 产品定位 | 多功能管理后台 | 简洁、本地优先的个人助手 |
| 北极星 | 功能覆盖度 | 打开后先回答「今天做什么」 |
| 默认导航 | 9 个 Tab 全展开 | 5 个核心 Tab + 按需启用模块 |
| 工程形态 | 单文件 HTML（3047 行） | 模块化源码 + 测试 + 构建脚本 + 便携版 |
| 可测试性 | 无单测 | 3 个领域测试套件 |

## 二、信息架构重构（最大改动）

### 2.1 导航收敛

**v3 导航**（9 个并列 Tab，平铺）：
```
📊 概览 · 📈 复盘 · 💼 工作 · 🔬 科研 · 🌿 生活 · 🏃 运动 · 🔥 习惯 · 💰 金融 · 📅 日历
```

**v4 导航**（三段两行布局 + sticky 毛玻璃）：
- **核心区**：☀️ 今天 · 💼 工作 · 📅 日历 · 📈 回顾
- **置顶区**：用户最多固定 3 个可选模块（动态生成）
- **更多**：☷ 进入模块中心

### 2.2 模块按需启用

新增 `src/app/module-preferences.js`，统一管理可选模块：

- **核心模块**：`overview / work / calendar / review / more`（始终可见）
- **可选模块**（默认隐藏）：
  - 专业场景：🔬 科研、💰 财务
  - 生活管理：🌿 生活、🏃 健康与运动
  - 效率管理：🔥 习惯
  - 信息工具：📰 信息热榜（默认关闭，避免打断专注）
- 关闭模块**只隐藏入口，不删除数据**
- 顶部最多固定 3 个可选模块，可在「更多」中调整顺序

### 2.3 「更多」页升级为模块中心

`src/ui/pages/more-page.js` 把原来零散的设置页改造成模块中心：

- 按分组（专业场景 / 生活管理 / 效率管理 / 信息工具）展示模块卡片
- 每张卡片显示：图标、名称、状态（已置顶 / 已启用 / 未启用）、已有数据量、描述
- 操作：启用 / 关闭、☆ 固定到顶部、← → 调整顺序、打开
- 同时收纳数据工具：导出备份、导入数据、本地快照、云端同步、快速命令、操作手册

### 3.2 三层架构：Page / Module / Selector

- **Page 层**：`ui/pages/*` 负责把 view-model 渲染成 HTML 片段
- **Module 层**：`app/module-registry.js` 注册渲染函数，找不到时回退到 legacy
- **Selector 层**：`app/selectors.js` 提供 `uiState() / filteredItems() / workModuleModel()` 等派生数据入口，避免页面直接散落读取全局状态

### 3.3 UI Helper Kit

`ui/helpers/panel-kit.js` 提供 `section / toolbar / metaRow / list / badge / summaryGrid / summaryCard / chips / empty` 等拼装能力，work / research / sport 页都改成 helper 驱动，减少直接拼 HTML 字符串。

### 3.4 渐进式迁移策略（Phase 3 → 9）

`bootstrap-phase3.js` 到 `bootstrap-phase9.js` 分阶段接管旧实现：

- 优先「包裹式接管」而不是一次性重写，确保功能不断裂
- legacy 层保留，做兼容兜底
- 已模块化接管的主干：Overview / Work / Finance / Research / Life / Sport / Habit

### 3.5 测试覆盖

| 测试文件 | 覆盖点 |
|---|---|
| `tests/module-preferences.test.js` | 模块启用、导航拦截、置顶上限、数据保留 |
| `tests/research-summary.test.js` | 论文阶段统计、等待/跟进优先级、跨类型截止日汇总 |
| `tests/health-metrics.test.js` | 个性化目标、周进度、跳过计划、身体趋势计算 |

### 3.6 一键构建便携版

```bash
python scripts/build_portable.py
```

按 `ORDER` 列表把 49 个 JS 模块拼成单个 `app.js`，连同 `app.css` 和 HTML 外壳一起输出到 `portable/`，普通用户仍然「点开一个 HTML 即可用」。

## 四、科研模块：从「记录工具」升级为「进展助手」

### 4.1 新增「科研首页」Tab

不再以「论文 / 项目 / 专利」三个 Tab 平铺作为首屏，而是先给一个**进展总览**：

- 4 张 summary card：📄 进行中论文、⚠️ 近期风险、🧭 缺少下一步、🏛️ 在研项目
- 成果管线 Pipeline：✍️ 撰写中 → 📨 投稿/审稿 → 🛠️ 修改中 → ✅ 已录用
- 「⏰ 最近需要处理」：聚合论文 / 专利 / 项目 / 事项的截止日，按风险等级（overdue / urgent / soon）高亮
- 「🧭 论文下一步」：列出 nextAction 或 waitingFor，按日期排序，一键打开编辑

### 4.2 论文模型增强

新增字段：

| 字段 | 用途 |
|---|---|
| `nextAction` | 下一步行动（如「补实验」「写 rebuttal」） |
| `nextDue` | 下一步截止日 |
| `waitingFor` | 等待对象（如「等导师反馈」「等合作者数据」） |
| `followUpAt` | 跟进日 |

行为联动：

- **阶段（status）与作者角色（kind）分离**：阶段走 `idea/draft/.../accepted`，角色走 `sub/collab/plan`，兼容旧版拟投/在投/合作数据
- **阶段变更自动写入历史**
- **论文下一步自动生成关联科研事项**（`sourceType: 'paper-action'`）；带日期时同步进入「今天」和日历
- 论文筛选 Tab 改为：进行中 / 撰写 / 投稿审稿 / 修改 / 已录用 / 已归档

### 4.3 跨类型截止日聚合

`domain/research/summary.js` 的 `deadlineItems()` 把以下四类截止日统一排序：

1. 论文：`nextDue / rebuttalDue / followUpAt` 中最早的一个
2. 专利：`feeDue`（缴费 / 答复）
3. 项目：`end`（结束 / 结题，排除已 closed）
4. 独立科研事项：`due`（排除自动生成的 paper-action）

## 五、健康模块：从「考核目标」转向「个性化 + 今天出发」

### 5.1 新增「健康首页」Tab

`domain/health/metrics.js` + `ui/pages/sport-page.js` 共同提供：

- **今天的计划**：完成并记录 / 改到明天 / 今天休息 / 恢复计划 四个动作
- **本周进度**：时长 + 次数两条进度条，按个人目标计算百分比
- **最近一次运动**：类型、时长、感受
- **身体趋势**：最新体重、近 7 天均值、近 30 天变化、目标体重

### 5.2 个性化目标（不再用固定考核）

新增 `data.prefs.healthGoals = { weeklyMinutes, weeklySessions }`：

- 默认 150 分钟 / 3 次，用户可随时调整
- 周回顾和今天页都使用个人目标，不再有「未达考核」的压迫感

### 5.3 运动计划更灵活

- **完成并记录**：自动用计划类型和时长填表，可调整实际时长和感受
- **改到明天**：把今天的计划顺延到明天
- **今天休息**：标记 `skipped`，保留原计划数据，可恢复
- **恢复计划**：取消 skipped 状态

### 5.4 运动记录表单收敛

v3 的表单字段较多，v4 收敛为：

- 类型、时长、日期、运动感受（轻松 / 刚好 / 吃力 / 接近极限）
- 名称可自动生成，不再强制填写

### 5.5 体重摘要重写

- v3：显示首次记录到最新的总差值，容易被首日波动放大
- v4：显示**近 7 天均值** + **近 30 天变化**，淡化单日波动
- 文案强调「关注一段时间的变化，不必被单日波动影响」

## 六、Bug 修复与细节优化

| 问题 | v4 修复 |
|---|---|
| 从今天 / 日历打开新建表单时分类偏移 | 保存打开表单时的分类 |
| 编辑任务时「里程碑」选中状态反转 | 修正判断逻辑 |
| 普通新建事项强制填入今天 | 仅从日历进入时预填日期 |
| 批量模式低频且占空间 | 移除批量模式，保留单条完成 / 编辑 / 删除 |

## 七、设计原则（v4 明确）

1. 打开后先回答「今天做什么」
2. 默认界面保持简单，专业能力通过模块按需开启
3. 启用和隐藏只改变入口，不破坏用户数据
4. 高风险的数据工具集中放在明确的数据中心（「更多」页底部）

## 九、迁移建议

1. **数据兼容**：v4 直接读取 v3 的 localStorage 数据，无需迁移
2. **首次打开**：默认只有 5 个核心 Tab，去「更多」启用你需要的模块即可
3. **常用模块**：建议把「科研」「健康」固定到顶部，符合博士后日常节奏
4. **构建自定义便携版**：修改 `source/src/` 后运行 `python scripts/build_portable.py` 重新生成
