# 个人工作台 v2 — 功能细化路线图（产品视角评审）

> 评审对象：`个人工作台v2.html`（单文件、本地优先、localStorage 存储的个人指挥中心）  
> 评审日期：2026-07-12 ｜ 角色：产品通

---

## 一、一句话定调

**当前版本已经是一个"完成度很高的记录器"，但还不是"教练"。**  
它把工作 / 科研 / 生活 / 运动 / 金融塞进了一个漂亮的单文件里，录入体验、跨模块日历、备份/同步都做得很扎实。真正的短板不在"能不能记"，而在：**记完之后有没有人帮你分析、提醒、复盘、形成习惯**。

产品信条：PM 是价值判断官，不是需求搬运工。下面按"投入产出比"排序给出细化建议。

---

## 二、已经做得很强（先肯定，避免瞎改）

- **跨模块统一日历 / 日程**：所有事项按日期落图，是真正的"统一指挥中心"。
- **科研模块建模专业**：论文（拟投/在投/合作 + 投稿时间线）、专利（生命周期）、科研项目（来源/角色/经费）三件套完整。
- **金融模块的净值计算**：基金持仓收益、区间涨幅、迷你走势图、收支趋势图、月度预算都齐了。
- **可靠性做得克制**：导出/导入 JSON、本地自动备份+回滚、GitHub Gist 同步、深色模式，该有的工程保障都有。
- **概览页已经有了"今日聚焦 / 连续记录 / 近期截止"的雏形**——这是向"教练"转型最好的支点。

---

## 三、核心判断：从"记录器"到"教练"的跃迁

| 维度   | 现在           | 目标                 |
| ---- | ------------ | ------------------ |
| 数据使用 | 录入 + 罗列      | 自动洞察 + 异常预警        |
| 触达方式 | 用户主动打开（pull） | 主动提醒（push）         |
| 时间视角 | 当日 / 当月      | 周复盘 / 季度 / 年度趋势    |
| 动机引擎 | 仅运动/体重有连续打卡  | 全场景连续性与成就体系        |
| 目标对齐 | 无目标层         | 年度目标 → 项目 → 任务 可追踪 |



---

## 四、按优先级的功能细化清单

> 优先级判定依据：用户价值 × 数据可支撑度 ÷ 实现成本。  
> 成本：**低**（改 UI/算法，数据结构不变）｜**中**（需加字段+迁移）｜**高**（新模块/新交互范式）。

### P0 — 低成本高价值，建议立刻做

**1. 财务「分类」治理 + 可视化（effort: 低~中）**

- *问题*：已有自由文本 `category` 和"分类汇总"列表，但① 无预置分类体系（"餐饮"和"吃饭"会分裂）、② 只有数字列表没有饼图/占比、③ 无"分类预算"、④ 缺**储蓄率**和\*\*资产结构（基金 vs 现金）\*\*指标。
- *细化*：预置可选分类 + 自由补充；分类支出环形图；分类级月度预算超支预警；概览新增"本月储蓄率"和"资产配置占比"。

**2. 全局「周复盘 / 周报」视图（effort: 低）**

- *问题*：个人工具最该有的"周日 10 分钟复盘"仪式完全没有。概览是"当下快照"，不是"周期回顾"。
- *细化*：新增一个视图，自动聚合——本周完成 X / 逾期 Y / 进行中 Z、下周待办、长期未动（停滞）事项、本周运动达标天数、本周收支与储蓄率。一键导出/可复制。

**3. 标签（tags）体系 + 搜索/筛选增强（effort: 中）**

- *问题*：`items` 只有 `cat + projectId`，无法做灵活分组（如"本周重点""等某人回复"）。搜索仅匹配标题/备注。
- *细化*：事项加 `tags[]`；全局搜索支持 `tag:`、`status:`、`project:` 过滤；各列表支持按标签筛。

**4. 习惯（Habits）成为一等公民模块（effort: 中）**

- *问题*：运动"周计划"只是习惯的弱代理；读书、冥想、记账等习惯无处安放。"统一指挥中心"的承诺在"日常习惯"上缺位。
- *细化*：独立 `habits` 模块，支持每日打卡、频率（每天/每周 N 次）、连续天数、与现有 streak 机制打通。

### P1 — 高价值，需数据结构改动

**5. 年度活动热力图 + 多维连续打卡（effort: 中）**

- *问题*：连续打卡只覆盖运动/体重，且概览没有"全局坚持感"的单一视图。
- *细化*：GitHub 式年度贡献图，按"当日是否有任意完成/记录"着色；新增任务完成、读书、记账等维度的 streak。

**6. 工作项目健康度 + 工时估算准确率（effort: 中）**

- *问题*：`projects` 只有 name/desc/status，无截止/负责人；`estH/actH` 已采集但**从不汇总**——你根本不知道自己估算准不准。
- *细化*：项目级"逾期任务数 / 风险里程碑"健康灯；按项目/总体汇总"估算 vs 实际"偏差率，作为个人效能指标。

**7. 目标 / OKR 层（effort: 高）**

- *问题*：缺"为什么做这些事"的目标锚点，任务与目标无法对齐。
- *细化*：年度/季度目标 → 关联项目/任务；概览展示"目标推进度"。这是把"工作台"升级为"指挥中心"的关键一层。

**8. 周期账单 / 订阅管理专属视图（effort: 中）**

- *问题*：`finances` 已有 `recur` 展开逻辑，但没有"我的订阅/固定支出"管理界面，也无"距下次扣费"提醒（专利年费有，会员费没有）。
- *细化*：订阅清单（金额/周期/下次扣费日）+ 临近提醒，与现有 recur 模板打通。

**9. 快捷捕获（全局快捷键 + 自然语言解析）（effort: 中）**

- *问题*：个人 capture 工具的核心体验是"秒记"。现在必须点对应模块→弹窗→填表。
- *细化*：`Cmd/Ctrl+K` 全局唤起；输入"买菜 tomorrow 18:00 #生活"自动解析标题/日期/标签/分类。

### P2 — 愿景 / 进阶

**10. 主动提醒（浏览器通知 + 每日摘要邮件/推送）**：从 pull 转 push，逾期/纪念日/扣费主动叫你。  
**11. 读书进度细化**：页码 % / 阅读时长，而不只是"在读"。  
**12. 轻量笔记 / 知识库**：可关联事项，capture 灵感。  
**13. 多设备同步冲突解决**：当前 Gist 同步是 last-write-wins（`__savedAt` 时间戳），并发编辑会丢数据；应做字段级合并或冲突提示。  
**14. 旅行预算 vs 实际消费联动 finance**：出行花钱自动归类到该旅行。  
**15. 科研"下一步动作"与审稿周期统计**：每篇论文记 next action；按期刊统计平均审稿时长、命中率。

## 五、挑 3 个最该先做的，附极简 PRD

### PRD-A：财务分类分析与可视化（P0-1）

- **用户故事**：作为*关注钱去向的用户*，我希望*看到分类支出占比和分类预算*，以便*控制不必要的开销*。
- **范围（Non-goals）**：不做自动银行流水导入；不接第三方记账 API。
- **成功指标**：分类支出环形图覆盖率 100%；用户连续 4 周设定≥1 个分类预算且超支被预警。

### PRD-B：周复盘视图（P0-2）

- **用户故事**：作为*想保持节奏的用户*，我希望*每周日看到自动生成的复盘*，以便*不遗漏停滞事项、规划下周*。
- **范围（Non-goals）**：不做 AI 自动写周报文案（可后续加）；不强制每日填写。
- **成功指标**：周复盘视图周打开率 ≥ 1 次/周（核心参与度指标）。

### PRD-C：习惯模块（P0-4）

- **用户故事**：作为*想培养习惯的用户*，我希望*在一个地方打卡所有日常习惯*，以便*形成连续坚持*。
- **范围（Non-goals）**：不替代运动周计划（二者可共存）；不做社交排行。
- **成功指标**：习惯模块上线后，全局"连续记录"天数中位数提升。

## 六、建议落地顺序

```
第 1 批（1~2 周，全低/中成本）：P0-2 周复盘 → P0-1 财务分类可视化 → P0-3 标签
第 2 批（2~4 周）：P0-4 习惯 → P1-5 热力图 → P1-6 项目健康/工时
第 3 批（按需）：P1-7 目标层 → P1-8 订阅 → P1-9 快捷捕获 → P2 系列
```

> 北极星指标建议：**"周活跃复盘率"**（每周至少打开一次周复盘/概览并标记≥1 项完成）。它同时衡量"数据被用起来"和"习惯在形成"，比单纯的功能数量更能说明产品在创造价值。
