# 个人工作台 v3 相对 v2 改进说明

> 角色：产品通（Product Management Expert）
> 一句话定位：**v2 是「记录器」，v3 开始成为「教练」**——核心变化不在"能不能记"，而在"记完之后有没有人帮你分析、提醒、复盘、养成习惯"。
> 北极星指标建议：`周活跃复盘率`（每周至少打开一次复盘并标记 ≥1 项完成）。它同时衡量"数据被用起来"与"习惯在形成"。

---

## 一、改进总览

| 模块 | v2 | v3 | 价值（Why） |
|------|----|----|------|
| 复盘 | 无（概览只是当下快照） | **新增「📈 复盘」页**，自动聚合本周完成/逾期/停滞/下周待办/运动达标/收支结余/习惯打卡 | 补齐"周日 10 分钟复盘"仪式感，把记录转成行动 |
| 习惯 | 仅"运动周计划"（弱代理） | **新增「🔥 习惯」一等模块**：每日/每周打卡、连续天数、近 7 天格子、CRUD | 读书/记账/冥想等无处安放的习惯有了归属 |
| 财务 | 有分类字段 + 列表式分类汇总 | **预置分类 + SVG 环形图占比 + 储蓄率卡 + 月度预算卡** | 从"知道花哪了"升级到"知道健不健康" |
| 标签 | 无（只有 cat + 项目维度） | **事项支持 tags**，列表显示 `#标签`，搜索支持 `tag:` 前缀 | 灵活分组（重点/等回复），跨项目检索 |
| 项目健康度 | 仅里程碑/任务进度条 | **健康灯（绿/黄/红）+ 逾期任务数 + 风险里程碑 + 工时估算准确率** | 一眼看出哪个项目在着火；用 est vs act 反哺排期能力 |
| 年度热力图 | 无 | **概览新增 GitHub 式半年度活跃热力图** | 用"连续天数"制造正反馈，强化坚持感 |
| 完成时间 | 仅状态标记 | 事项 `done` 时写入 `completedAt` | 复盘/热力图才有真实时间锚点（而非仅依赖 due） |

## 二、P0 四大改进详解

### 1. 📈 周复盘视图（最高杠杆）
- **用户故事**：作为用户，我希望每周日打开一页就能看到"本周完成了什么 / 什么逾期了 / 什么停滞了 / 下周要做什么 / 运动达标没 / 钱剩多少 / 习惯打了几回"，以便用 10 分钟做完复盘而不必自己翻各个模块。
- **实现**：`renderReview()` 按"本周一~本周日"窗口聚合；对比上周完成数（↑/↓）；自动识别"停滞风险"（无截止日且创建 >14 天，或有截止日且已过 -3 天以上）；直接复用 `itemHTML` 渲染明细列表。
- **Non-goals**：不做 AI 自动生成复盘文案、不做跨周趋势图（属后续迭代）。

### 2. 💰 财务分类可视化
- **预置分类**：表单 `category` 改为带 `datalist`（工资/理财收益/餐饮/房租/交通/购物/医疗/教育/娱乐/其他），避免"餐饮/吃饭"分裂。
- **环形图**：用 SVG `stroke-dasharray` 按支出占比画环，配同色图例 + 百分比。
- **新指标卡**：`储蓄率 = 结余 / 收入`（≥20% 绿、0~20% 黄、负红）；`本月预算` 进度条（结余/预算 + 剩余）。
- **Non-goals**：不做资产负债表、不做多账户、不做投资收益率计算。

### 3. 🏷️ 标签体系
- 事项表单加「标签（逗号分隔）」；卡片以 `#标签` 紫色 chip 显示。
- 搜索框支持 `tag:重点` 前缀语法，可与关键词共存（`filtered()` 内 `hay` 已并入 tags，`tag:` 走精确包含逻辑）。

### 4. 🔥 习惯模块
- 独立 `habits` 数据（`{id,name,freq:'daily'|'weekly',target,logs:{日期:true}}`）。
- 卡片展示：连续打卡天数、近 7 天格子（点击当天即打卡/取消）、本周次数与达标率（周频次模式）。
- 新建/编辑/删除表单 + 二次确认。

## 三、P1 两大改进详解

### 5. 年度活动热力图
- 概览底部 GitHub 式 26 周网格；当日有"完成事项 / 运动 / 记账 / 记体重 / 习惯打卡"任一即着色，强度分 4 档。
- 顶部显示"近半年活跃 X 天"。

### 6. 项目健康度 + 工时估算准确率
- 项目卡加**健康灯**：完成=绿；逾期≥3 或存在 7 天内风险里程碑=红；有逾期=黄。
- 显示「逾期 N」「⚠风险里程碑 N」。
- **工时估算准确率** `acc = actH/estH`：v2 已采集 `estH/actH` 但从不汇总，v3 首次把它变成可观测的个人效能指标。

## 四、数据模型变更（兼容性）

| 字段 | 说明 | 兼容处理 |
|------|------|----------|
| `data.habits` | 新数组 | `load()` / `restoreBak()` / 同步 / storage 事件四处均加 `if(!data.habits)data.habits=[]`，导入旧 v2 备份不丢数据 |
| `item.completedAt` | 完成时间戳 | `toggle()` 与 `submitForm()` 写入；旧数据无此字段时回退到 `due` 显示 |
| `item.tags` | 标签数组 | 旧数据无 tags，`(i.tags\|\|[])` 全部兜底为空 |

导入/导出 JSON、本地存储、多标签同步、主题、备份还原等 v2 能力**全部保留**，v3 为纯增量改造，v2 文件未改动。

## 五、验证情况（如实说明）

- ✅ **语法**：整段 `<script>` 解析通过（`SYNTAX OK`）。
- ✅ **静态一致性**：复盘/习惯/热力图所有 `onclick` 引用的函数均存在；所有 `getElementById` 的 id（`habitMask`/`h_name`/`f_tags`/`finCats` 等）均在 HTML 中定义；新函数依赖的辅助函数（`mondayOf`/`ymdL`/`daysBetween`/`esc`/`itemHTML`/`uid`）均已在 v2 中存在。
- ✅ **逻辑自洽**：render 分发、nav 视图名（`habit` 单数 vs 数据 `habits` 复数）、财务 type 值（`income`/`expense`）全部对齐，无断链。
- ⚠️ **未做真机运行**：浏览器自动化技能当前不可用，jsdom 安装被本机代理拦截，故未做真实 DOM 渲染跑测。**建议你直接双击打开 v3 文件点一遍**：复盘页、习惯页（建一个习惯并打卡）、财务页（看环形图与储蓄率）、概览底部热力图。如发现任何运行时报错，把控制台信息发我即可秒修。
