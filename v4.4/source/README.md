### 网盘文件夹备份使用说明

#### 它是怎么工作的

工作台不会直接登录网盘，也不需要填写网盘账号或 Token。它会把备份 JSON 写入电脑上的网盘同步目录，再由 OneDrive、iCloud Drive、Dropbox、百度网盘等客户端负责上传：

```text
个人工作台 → 电脑上的同步文件夹 → 网盘客户端 → 云端
```

因此，使用前需要先安装并登录相应的网盘电脑客户端，并确认所选目录正在同步。

#### 第一次备份

1. 使用桌面版 Chrome 或 Edge 打开 `portable/个人工作台.html`。
2. 进入“更多 → 数据与工具 → 网盘文件夹备份”。
3. 点击“选择网盘文件夹”。
4. 在系统窗口中选择正在同步的网盘目录。建议在网盘内先新建一个“个人工作台”专用文件夹，再选择它。
5. 允许文件夹访问。工作台会在其中创建“个人工作台备份”子文件夹并立即生成第一份备份。

备份目录中会包含：

- `个人工作台_最新.json`：始终指向最近一次成功备份。
- `个人工作台_YYYY-MM-DD_HHMMSS.json`：带时间的历史版本，最多保留最近 30 份。

开启“每天自动备份”后，工作台会在打开页面和保存数据时检查，24 小时内最多自动生成一份；也可以随时点击“立即备份”。

#### 在另一台电脑恢复

1. 先安装并登录同一个网盘客户端，等待备份文件夹同步完成。
2. 用 Chrome 或 Edge 打开另一台电脑上的个人工作台。
3. 进入“网盘文件夹备份”，选择同一个网盘同步目录。
4. 如果检测到已有历史，工作台会暂停自动备份，避免新设备上的空数据或示例数据覆盖“最新版本”。
5. 在“可恢复版本”中确认日期并点击“恢复”。
6. 检查数据无误后，再开启“每天自动备份”。

#### 安全与常见问题

- 恢复前会自动创建一份本地快照，防止误覆盖后无法撤回。
- 点击“断开”只会取消工作台与文件夹的连接，不会删除网盘里的备份。
- 这项功能是“定期备份与手动恢复”，不会自动合并两台电脑同时产生的修改；需要实时跨设备同步时仍可使用 GitHub Gist。
- 如果点击“选择网盘文件夹”没有反应，说明当前浏览器不支持目录直写，请换用桌面版 Chrome 或 Edge。
- 如果重新打开后提示“需要重新授权”，点击“立即备份”并再次允许文件夹访问即可。
- 网盘客户端未启动、目录未设置为同步，或云端空间已满时，文件可能只保存在电脑上，请到网盘客户端确认上传状态。

## v4.4 新增

- 信息热榜改为“今日精选、按来源看、稍后阅读”三个入口，默认不再铺满整页热榜

- 搜索会同时作用于今日精选和按来源视图，收藏内容不依赖来源当前是否启用

## 产品原则

- 打开后先回答“今天做什么”
- 默认界面保持简单，专业能力通过模块按需开启
- 启用和隐藏只改变入口，不破坏用户数据
- 高风险的数据工具集中放在明确的数据中心

## 保持的设计原则

- `portable/个人工作台.html` 仍是普通用户入口
- `source/` 仍是开源和持续重构用源码
- legacy 层仍保留，用于渐进式迁移，不做一次性推翻
- 优先“包裹式接管”而不是一次性重写，确保功能不断裂
- 先建立 page / module / selector 三层入口，再逐步把页面改造成 view-model 驱动
