# 个人工作台 · v4 → v4.1 升级说明

> v4.1 延续「简洁、本地优先的个人助手」方向，把**财务模块**从「记录列表」升级为「可行动的个人财务助手」。默认导航不变，仍只保留 今天 / 工作 / 日历 / 回顾 / 更多 五个核心 Tab。

## 一、财务模块：从「记账」到「回答这个月怎么办」

### 1.1 四入口拆分

v4 的财务是单一列表 + 简单汇总。v4.1 拆分为四个清晰入口：

| 入口 | 用途 |
|---|---|
| **财务首页** | 优先回答：本月收入、支出、结余、预算余量、主要分类、未来固定账单 |
| **收支明细** | 单条记录的录入与查询，表单收敛为「类型 / 金额 / 日期 / 分类」，低频字段收进「更多信息」 |
| **预算与账单** | 总预算 + 分类预算设置，展示已用、剩余和超支状态 |
| **基金** | 分为「持仓」与「仅关注」，汇总市值、持仓收益，并提示净值数据新鲜度 |

### 1.2 统一「实际发生」口径（核心改动）

v4 的财务会把周期账单提前计入收支和资产，导致：
- 本月预算看起来已经超支，但其实账单还没真正发生
- 趋势图被「假数据」拉偏

v4.1 改为：

- **未来记录和周期账单不提前计入收支、趋势和资产**
- 周期收支变成「待确认账单」：到期后可**确认入账**、**调整来源规则**或**忽略本次**
- 资产数字真实反映当下，不再被未来账单预扣

### 1.3 预算系统升级

- **总预算**：可设置本月总预算
- **分类预算**：每个分类独立设上限
- 状态展示：已用 / 剩余 / 超支，超支会有提示

### 1.4 基金模块改进

- 区分「持仓」（影响资产）和「仅关注」（仅看净值）
- 汇总市值、持仓收益
- **净值数据新鲜度提示**：避免拿过期净值算资产

## 二、表单收敛

v4 的收支录入表单字段较多。v4.1 收敛为：

- **核心字段**：类型、金额、日期、分类
- **低频字段**：收进「更多信息」展开区

降低日常记录的认知负担。

## 三、数据兼容

- 完全兼容 v4 的收支、预算和基金记录
- 老数据无需迁移，直接打开 v4.1 即可使用

## 四、测试覆盖

新增 `tests/finance-metrics.test.js`，覆盖：

- 实际与计划分离（核心改动）
- 总预算与分类预算计算
- 分类汇总与趋势统计
- 基金持仓与仅关注区分

至此 v4.1 共有 4 个测试套件：

| 测试文件 | 覆盖点 |
|---|---|
| `tests/module-preferences.test.js` | 模块启用、导航拦截、置顶上限、数据保留 |
| `tests/research-summary.test.js` | 论文阶段统计、等待/跟进优先级、跨类型截止日汇总 |
| `tests/health-metrics.test.js` | 个性化目标、周进度、跳过计划、身体趋势计算 |
| `tests/finance-metrics.test.js` | 实际与计划分离、预算、分类及趋势统计 |

## 五、迁移建议

1. **数据兼容**：v4.1 直接读取 v4 的 localStorage 数据，无需迁移
2. **首次打开**：财务模块入口位置不变，老用户无缝过渡
3. **首要体验**：去「财务首页」看看本月预算余量和未来固定账单——这是 v4 没有的视角
4. **构建自定义便携版**：修改 `source/src/` 后运行 `python scripts/build_portable.py` 重新生成
